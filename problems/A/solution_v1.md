# A题初步思路 v1

## 1. 题目理解

A 题的核心任务不是一般意义上的“瓶子检测”，而是对**透明塑料饮料瓶内是否有残留液体**进行稳定识别，并进一步推广到：

1. 已知瓶体区域时，判断是否有液体。
2. 在不同瓶型、姿态、液体类型、背景和光照下保持泛化能力。
3. 进一步实现自动定位瓶体，并把液体含量粗分为少量、中等、较多等类型。

题目还给了几个很重要的约束：

- 每张图只有一个瓶子。
- 瓶子整体清晰、形状基本完整、无遮挡严重。
- 姿态可以任意，包括竖放、横放、倾斜、倒放。
- 数据集至少 100 张，且生成式图片不宜作为主体。

这意味着本题最合理的路线不是追求“超大规模通用检测”，而是围绕**单瓶场景、透明边界、液体弱纹理、姿态变化**四个难点做一个有解释性的视觉模型。

## 2. 对“以 LCDTC 为主体，加入 Trans2Seg 思想”的判断

我的判断是：**这个方向是对的，但不能简单把 Trans2Seg 整体替换到 LCDTC 上，最好采用“LCDTC 主体 + Trans2Seg 思想做分割增强”的方式。**

原因如下。

### 2.1 为什么 LCDTC 适合作为主体

`Wu et al. - 2023 - Liquid Content Detection In Transparent Containers A Benchmark` 与本题最接近：

- 任务本身就是透明容器 + 液体含量联合识别。
- 标签体系天然可迁移：`empty / little / half / much / full`。
- 数据里包含透明容器、液体类别粗分、检测框标注，和题目目标高度一致。
- 它的基线 LCD-YOLOF / LCD-YOLOX 提供了一个可复现、可解释、可扩展的起点。

因此，LCDTC 最适合作为我们的**预训练任务、标签定义参考和整体建模框架**。

### 2.2 为什么 Trans2Seg 的思想值得加

`Xie et al. - 2021 - Segmenting Transparent Object in the Wild with Transformer` 的关键价值不在于“又一个分割模型”，而在于它抓住了透明目标的本质难点：

- 透明目标和背景局部纹理相似，单靠局部卷积特征不够。
- 透明边界依赖全局上下文和长距离关系。
- 基于 prototype/query 的解码方式，对“边界弱、区域细、反光多”的目标更稳定。

这与本题高度一致。瓶子外轮廓、可见透明区域、液面附近的纹理变化，本质上都不是强纹理目标。只做 bbox 分类，模型很容易学到背景偏差、标签偏差或瓶贴颜色偏差，而不是“液体真实存在”的视觉证据。

### 2.3 为什么不能直接照搬 Trans2Seg

直接把 Trans2Seg 作为主模型有三个问题：

1. 本题主任务不是纯语义分割，而是“检测 + 存在判断 + 粗量级分类”。
2. 我们当前最强的公开近邻数据是 LCDTC，它主要是检测框标签，不是高质量像素级 mask 标签。
3. 比赛数据量不会很大，直接上完整 Transformer 分割架构，容易训练不稳、推理偏慢、实现成本偏高。

因此，更合理的做法是：

- **保留 LCDTC 风格的检测/分类主线。**
- **把 Trans2Seg 的全局建模和 prototype 解码思想加到瓶内区域理解上。**
- **把“分割结果”作为液体判定的辅助证据，而不是唯一输出。**

## 3. 建议的总体路线

建议采用一个**两阶段、三任务联合**的方案：

### 阶段 A：瓶体定位

目标：

- 输入整图。
- 输出瓶体 bounding box。

实现建议：

- 以 LCD-YOLOX 或 YOLOX-S 风格检测器为主。
- 在自建数据集上只保留“饮料瓶”这一类，降低检测复杂度。
- 如果比赛第二问给了人工标注瓶体区域，则这一阶段可跳过，直接使用标注框进入阶段 B。

### 阶段 B：瓶内液体理解

目标：

- 输入瓶体 ROI。
- 输出：
  - 是否有液体（二分类）
  - 粗量级类别（空 / 少量 / 中等 / 较多）
  - 可选的瓶体 mask 与液体 mask

核心思路：

- 用 LCDTC 的液体状态分类作为主监督。
- 用 Trans2Seg 风格的 segmentation branch 学习：
  - 瓶体区域
  - 瓶内可见液体区域
  - 背景/反光干扰

### 阶段 C：分类与几何证据融合

单纯依赖分类 logits 不够稳，建议引入由 mask 导出的几何特征：

- `r1 = 液体区域面积 / 瓶体可见区域面积`
- `r2 = 液体区域在瓶体内部的重心位置`
- `r3 = 液体区域的连通性、边界平滑度`
- `r4 = 与瓶体主轴方向相关的姿态特征`

最终使用“深度特征 + 分割特征 + 几何特征”融合判断液体状态。

## 4. 一个更合适的模型形态

我建议的初版模型可以命名为：

`LCDTC-Trans2Seg-BottleNet`

结构如下。

### 4.1 主干网络

可选两种做法：

1. 继续沿用当前 LCDTC 复现代码的 YOLOX/CSPDarknet 主干。
2. 若后续实验显示透明边界建模不足，再换成更强的 ConvNeXt-T / Swin-T 级别主干。

v1 建议保守一些，先沿用现有 YOLOX 路线，因为：

- 你已经有 LCDTC 代码基础。
- 工程量最小。
- 先把任务链条跑通更重要。

### 4.2 ROI 级 Trans2Seg 风格分支

对检测框裁剪后的瓶体 ROI，增加一个轻量的 Transformer 分割头：

- 编码器输入：瓶体 ROI 特征图。
- 解码器 query：少量可学习 prototype。
- prototype 建议不是按 11 类透明物体设置，而是按本题重定义：
  - bottle
  - liquid
  - background
  - reflection/noise

这样做的好处是：

- 任务定义更贴近比赛。
- query 数量少，训练更稳。
- 分割结果能直接服务于液体存在判断。

### 4.3 状态分类头

在 ROI 级特征上增加两个分类头：

1. `binary head`：空 / 非空
2. `coarse head`：空 / 少量 / 中等 / 较多

如果想最大化利用 LCDTC 标签，可以先训练 5 类：

- empty
- little
- half
- much
- full

然后在比赛报告中再映射为：

- 空：`empty`
- 少量：`little`
- 中等：`half`
- 较多：`much + full`

这样既保留细粒度信息，也方便和 LCDTC 对齐。

### 4.4 几何一致性分支

为了让分割不是“画着玩”，建议加入一个简单但很有价值的约束：

- 若分类为 `empty`，则 liquid mask 面积应接近 0。
- 若分类为 `little / half / much / full`，则 liquid mask 面积比例应落在对应区间。

例如可设置软约束：

- `empty`: `r1 ≈ 0`
- `little`: `r1 ∈ [0.02, 0.20]`
- `half`: `r1 ∈ [0.20, 0.55]`
- `much/full`: `r1 > 0.55`

这个区间不需要一开始就很精确，可以根据验证集再调。

## 5. 数据方案

## 5.1 数据来源结构

建议把数据分成三层：

### 第一层：LCDTC 公开数据

用途：

- 做预训练。
- 学习“透明容器 + 液体状态”的通用表示。

注意：

- LCDTC 里的容器不只是一种塑料饮料瓶。
- 它更适合做**源域预训练**，不能直接当比赛最终分布。

### 第二层：自建比赛目标数据

这是主数据集，必须围绕题目本身采集。

建议至少做到：

- 300 到 800 张为宜，100 张只是最低线。
- 尽量覆盖：
  - 矿泉水瓶、饮料瓶、茶饮瓶、果汁瓶、碳酸饮料瓶
  - 有标签/无标签
  - 有瓶盖/无瓶盖
  - 透明 / 半透明
  - 清水、茶色液体、有色液体、浑浊液体
  - 空、少量、中等、较多
  - 竖放、横放、倾斜、倒放
  - 室内自然光、顶光、侧光、复杂背景

### 第三层：透明目标分割辅助数据

若后续需要增强分割能力，可引入透明物体分割数据做辅助训练。

用途：

- 不是为了直接分类液体。
- 而是为了提升“透明边界、玻璃/塑料轮廓、弱对比区域”的感知能力。

这部分正是 Trans2Seg 思想最适合发挥作用的地方。

## 5.2 标注建议

最低标注：

- 瓶体 bbox
- 二分类标签：空 / 非空

更推荐的完整标注：

- 瓶体 bbox
- 瓶体 mask
- 液体 mask
- 液体状态标签：empty / little / half / much / full
- 姿态标签：upright / horizontal / tilted / inverted

其中：

- `瓶体 mask` 能明显帮助透明目标分割。
- `液体 mask` 是后续做可解释分析和几何融合的关键。
- `姿态标签` 能缓解“同样液体量、不同放置姿态视觉差异巨大”的问题。

## 6. 训练策略

建议分三步走。

### 第一步：LCDTC 预训练

在 LCDTC 上训练 5 类液体状态检测模型，目标是获得：

- 对透明容器的基础检测能力
- 对液体存在与粗量级的初始判别能力

### 第二步：自建瓶子数据微调

在比赛自建数据上微调：

- 检测头：适配单类饮料瓶
- 分类头：适配空/非空与粗量级
- 分割头：适配瓶体与液体 mask

### 第三步：联合优化

多任务损失可设为：

`L = λ1 L_det + λ2 L_seg_bottle + λ3 L_seg_liquid + λ4 L_cls_bin + λ5 L_cls_state + λ6 L_consistency`

其中：

- `L_det`：检测损失
- `L_seg_bottle`：瓶体 mask 分割损失
- `L_seg_liquid`：液体 mask 分割损失
- `L_cls_bin`：空/非空分类损失
- `L_cls_state`：粗量级分类损失
- `L_consistency`：分类与面积比例之间的一致性约束

分割损失建议使用：

- `BCE + Dice`

分类损失建议使用：

- `Focal Loss` 或带类别权重的交叉熵

因为空瓶和少量液体样本很可能不均衡。

## 7. 为什么这条路线比“纯检测分类”更稳

如果只做 bbox 级分类，模型容易依赖一些伪特征：

- 背景颜色
- 瓶贴颜色
- 拍摄场景
- 常见瓶型偏差
- 反光或阴影

加入 Trans2Seg 风格的 ROI 分割后，模型被迫回答两个更本质的问题：

1. 这个瓶子的透明主体在哪里？
2. 这个瓶子内部哪些区域更像真实液体，而不是标签、反光或背景穿透？

这能显著提升模型的解释性，也更符合论文写作需要。

## 8. 与题目三个问题的对应关系

### 问题 1：构建数据集

对应方案：

- 自建“透明塑料饮料瓶残液数据集”
- 以真实拍摄为主体，公开图片为补充
- 提供来源、类别分布、姿态分布、含量分布

### 问题 2：已知瓶体区域，判断是否有液体

对应方案：

- 直接使用瓶体 ROI 输入阶段 B 网络
- 主输出为空 / 非空
- 辅助输出为液体分割与粗量级

### 问题 3：自动定位 + 粗量级分类

对应方案：

- 加入阶段 A 检测器实现自动定位
- 阶段 B 完成液体状态分类
- 最终输出：
  - 瓶体框
  - 空 / 非空
  - 少量 / 中等 / 较多

## 9. 评价指标建议

题目没有强行指定指标，因此建议分任务给指标。

### 二分类任务

- Accuracy
- Precision
- Recall
- F1-score

其中 F1 比单纯 Accuracy 更重要，因为空瓶和非空瓶通常会不平衡。

### 检测任务

- mAP@0.5
- mAP@0.5:0.95

### 粗量级分类任务

- Macro-F1
- Confusion Matrix

### 分割辅助任务

- IoU
- Dice

在论文写作上，建议把“二分类性能”作为主结果，把“分割与粗量级结果”作为解释性和扩展性证据。

## 10. 预期实验设计

建议至少做四组对比实验。

### 实验 1：纯分类基线

- 输入瓶体 ROI
- 直接用 CNN/ViT 做空/非空分类

意义：

- 给出最低可行基线

### 实验 2：LCDTC 风格检测/分类

- 用 LCDTC 式检测框 + 状态分类

意义：

- 验证公开任务到比赛任务的迁移效果

### 实验 3：加入 Trans2Seg 风格分割分支

- 在实验 2 基础上增加 bottle/liquid segmentation branch

意义：

- 验证全局透明边界建模是否有效

### 实验 4：加入几何一致性融合

- 在实验 3 基础上融合液体 mask 比例与姿态信息

意义：

- 验证“分类 + 分割 + 几何”的联合收益

如果实验设计合理，这四组对比能很自然支撑整篇论文的技术路线。

## 11. 风险与应对

### 风险 1：自建数据量偏小

问题：

- Transformer 风格分支对数据量更敏感。

应对：

- 先用 LCDTC 预训练。
- 将 Trans2Seg 思想只放在轻量 ROI 头中，而不是全模型重构。

### 风险 2：液体 mask 标注成本高

问题：

- 精细像素标注耗时。

应对：

- 先对 100 到 200 张核心样本做高质量 mask 标注。
- 其余样本只做 bbox + 状态标签。
- 后续用半监督或伪标签扩展。

### 风险 3：不同姿态下“含量”和“视觉占比”不一致

问题：

- 横放瓶和竖放瓶，液体投影面积不能直接等价。

应对：

- 加姿态标签或姿态分支。
- 面积比例只作为辅助证据，不作为唯一依据。

### 风险 4：公开数据与比赛分布不一致

问题：

- LCDTC 不完全等于透明塑料饮料瓶残液任务。

应对：

- 把 LCDTC 作为源域预训练，而不是最终结论数据集。
- 最终结果必须以自建比赛数据测试为主。

## 12. 当前最推荐的执行版本

如果现在就开始落地，我建议采用下面这个版本作为 v1：

1. 用 LCDTC 训练一个 5 类液体状态基线模型。
2. 自建一个“透明塑料饮料瓶残液数据集”，优先保证样本真实、多姿态、多液体类型。
3. 在瓶体 ROI 上增加一个轻量 Trans2Seg 风格分割头，输出 bottle mask 和 liquid mask。
4. 以“空/非空”为主任务，以“粗量级分类”为副任务，以“mask 几何比例一致性”为辅助约束。
5. 最终形成一个可解释的两阶段系统：`检测瓶子 -> 理解瓶内液体`。

这条路线的优点是：

- 与题目目标对齐。
- 与 LCDTC 文献主线一致。
- 能合理吸收 Trans2Seg 的核心思想。
- 工程上可渐进实现，不需要一次推倒重来。

## 13. 一个需要特别注意的小问题

当前 `problems/A/data` 里有一篇：

- `Zhang and Yan - 2022 - Crossformer Transformer Utilizing Cross-Dimension Dependency for Multivariate Time Series Forecasting`

这是一篇**时间序列预测论文**，不是透明目标检测/分割的视觉骨干论文。它和 LCDTC 论文中提到的视觉 `CrossFormer` 不是同一条技术线，后续写作和实现时不要把这篇当作视觉 backbone 依据。

## 14. 结论

总结来说：

- **以 LCDTC 为主体是合理的。**
- **加入 Trans2Seg 思想也是合理的。**
- 但最佳方案不是“把 LCDTC 改成纯 Trans2Seg”，而是：
  - **用 LCDTC 提供任务框架和预训练基础**
  - **用 Trans2Seg 提供透明边界与瓶内区域理解能力**
  - **再结合几何与姿态信息，做一个面向单瓶残液识别的多任务系统**

我认为这会是一条既能写成论文、又能真正做出效果的路线。
