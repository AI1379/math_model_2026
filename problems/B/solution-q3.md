# 问题3：小组赛比赛地点选择 —— 算法综述与比较

## 1. 问题形式化

**已知**：浙江省 64 个参赛单位（11 个地级市代表队 + 53 个县区代表队），已按 Q1 方案分成 16 个小组，每组 4 队。

**要求**：从 64 个单位中选 8 个作为比赛场地，每个场地承办 2 个小组。目标是最小化所有队伍到其对应场地的总旅行距离（Haversine 大圆距离），同时兼顾地理覆盖的公平性。

**数学形式** (p-median + 容量约束)：

$$
\begin{aligned}
\min \quad & \sum_{g=1}^{16}\sum_{k=1}^{64} D(g,k) \cdot z_{g,k} \\
\text{s.t.} \quad & \sum_{k=1}^{64} y_k = 8 \quad & \text{(选8个场地)} \\
& \sum_{k=1}^{64} z_{g,k} = 1 \quad \forall g \quad & \text{(每组恰好指派到1个场地)} \\
& z_{g,k} \le y_k \quad \forall g,k \quad & \text{(只能指派到已选场地)} \\
& \sum_{g=1}^{16} z_{g,k} = 2y_k \quad \forall k \quad & \text{(每场地恰好2组)} \\
& y_k \in \{0,1\},\; z_{g,k} \in \{0,1\}
\end{aligned}
$$

其中 D(g,k) = 组 g 中 4 支球队到场地 k 的距离之和。变量规模：64 个选址变量 + 16×64 个指派变量 = 1088 个二元变量。

## 2. 已实现算法

### 2.1 ILP 联合优化（当前主要方法）

**文件**：[q3_venue.py](q3_venue.py) 中 `ilp_venue_selection()`

**思路**：将选址和指派联合建模为整数线性规划，用 PuLP 调用 COIN-OR CBC 求解器精确求解。

**目标**：最小化团队到场地总旅行距离（按队累加，而非按组累加）。

**约束**：
- 恰好选 8 个场地
- 每个小组恰好指派到 1 个场地
- 只能指派到已选场地
- 每个已选场地恰好承办 2 个小组

**规模**：64 个 y_k 变量 + 16×64 个 z_{g,k} 变量 + 约 60 个约束（不含上界）。

**时间复杂度**：CBC 在 60 秒时间限制内能找到精确最优解（实际通常收敛更快）。

**结果**：总旅行距离 7617 km，平均每队 119.0 km，最远单队 337 km。

**优点**：
- 精确最优（给定时间限制内）
- 建模清晰，约束易于扩展
- PuLP 代码量小

**缺点**：
- 依赖第三方求解器 (PuLP + CBC)
- 规模扩大时可能组合爆炸
- 无法提供最优性 gap 的理论保证（CBC 内部计算但用户不可控）

### 2.2 启发式贪心选址 + ILP 指派

**文件**：[q3_venue.py](q3_venue.py) 中 `heuristic_venues()` + ILP 指派

**思路**（两阶段）：

**阶段一**：贪心选址
1. 计算每个地点到所有 64 队的距离和，作为"成本"
2. 选择成本最低的地点作为第一个场地
3. 迭代 7 轮：每轮选一个地点，综合考虑"距离成本低"和"离已选场地最远"（保证地理分散）
4. 评分函数：`score = minDistToSelected / (totalCost + 1)`

**阶段二**：给定 8 个场地后，用 ILP 求解最优小组-场地指派（保证每场地恰好 2 组）。

**结果**：总旅行距离 9962 km，比 ILP 最优多 30.8%。

**优点**：
- 地理覆盖更均匀（经纬度跨度更大）
- 选址阶段不依赖 ILP，可纯贪心
- 可以灵活修改评分函数

**缺点**：
- 选址和指派分离，非联合优化
- 贪心容易陷入局部最优
- 对初始点的选择敏感

### 2.3 广义蒙特卡洛验证

**文件**：[q3_q4_generalized.py](q3_q4_generalized.py) + [generalized_mc.py](generalized_mc.py)

**思路**：不限于浙江省数据，在随机生成的"架空省份"上验证选址效果的鲁棒性。
- 随机生成 150 个不同拓扑结构（城市数、县级队分布）
- 每个拓扑上跑多次贪心选址
- 分析约束紧度 τ 与选址效果的相关性

**结论**：选址效果（总距离、最大单队距离）与约束紧度 τ 无显著相关性（相关系数接近 0），说明分组层面的约束不会系统性劣化选址。

## 3. 替代算法（未实现）

### 3.1 拉格朗日松弛 (Lagrangian Relaxation)

#### 原理

将"每个场地恰好 2 组"的容量约束松弛到目标函数中，用拉格朗日乘子 λ_k 惩罚违反量：

约束的原问题 (P) 转化为拉格朗日对偶问题 (D)：

$$
L(\lambda) = \min_{y,z} \left\{ \sum_{g,k} D(g,k)z_{g,k} + \sum_k \lambda_k\left(\sum_g z_{g,k} - 2y_k\right) \right\}
$$

整理后：

$$
L(\lambda) = \min_{y,z} \left\{ \sum_{g,k} \big(D(g,k) + \lambda_k\big) z_{g,k} - 2\sum_k \lambda_k y_k \right\}
$$

约束只剩：
- Σ y_k = 8
- Σ_k z_{g,k} = 1（每组去一个场地）
- z_{g,k} ≤ y_k（只能去已选场地）

**关键性质**：对于任意 λ，L(λ) ≤ 原问题最优值。所以 L(λ) 是原问题的**下界**。

#### 子问题求解

给定 λ 后，问题分解为：
- 对每个组 g：选择"修正距离"最小的已选场地（组之间独立！）
- 全局：选哪 8 个场地能最小化 Σ_g（最小组内场地修正距离）

8 个场地来自 64 个候选，组合数 C(64,8) ≈ 4.4×10^9，但可以用动态规划或局部搜索近似。

#### 次梯度优化算法

```
输入: 距离矩阵 D(g,k), 最大迭代次数 T
输出: 原问题最优值的下界, 可行解

1. 初始化 λ_k = 0, k=1..64
2. 初始化最优下界 LB = -∞
3. 初始化步长参数 α_0

4. for t = 1 to T:
   4.1 求解子问题 LR(λ)，得到 (y*, z*)
   4.2 L = 目标函数值
   4.3 更新最优下界: LB = max(LB, L)
   4.4 计算次梯度: γ_k = Σ_g z*_{g,k} - 2·y*_k    // 违反量
   4.5 若 ||γ|| < ε: 收敛，跳出
   4.6 更新步长: α_t = α_0 / t   (或 Polyak 步长)
   4.7 更新乘子: λ_k ← λ_k + α_t · γ_k
   4.8 尝试从松弛解构造可行解 → 更新上界 UB

5. 返回 LB, UB, optimality_gap = (UB-LB)/UB
```

#### 从松弛解构造可行解

松弛解 z* 可能违反容量约束（某场地被分配了 ≠2 个组，或分配到了未选的场地）。

修复策略：
1. 保留 y* 中选中的场地（不够 8 个则补选成本最低的场地）
2. 对"超容量"的场地（>2 组），保留距离最小的 2 组，其余重新指派到有"空位"的最近场地
3. 对"空场"（0 组），从其他场地移入距离最近但非最优的组

#### 复杂度分析

- 每次子问题：O(64 × 2^64) 如果用枚举；但用贪心近似可降至 O(64²)
- 次梯度迭代：通常 100-500 轮收敛
- 总复杂度：O(T × n²)，远低于 ILP 的组合搜索

#### 优势与局限

| 优势 | 局限 |
|------|------|
| 提供理论下界，可量化解的最优性 | 子问题仍有一定复杂度 |
| 问题分解后，各组独立，可并行 | 对偶间隙可能非零（整数规划） |
| 乘子 λ_k 有经济解释（场地的"影子价格"） | 需要手写迭代逻辑，无现成求解器 |
| 规模扩大时优势更明显 | 步长和迭代次数需调参 |

**λ_k 的解释**：λ_k > 0 表示场地 k 当前"过于拥挤"，需要提高 k 的使用成本来"劝退"部分组；λ_k < 0 表示场地 k "空置"，需要降低 k 的使用成本吸引更多组。

### 3.2 K-medoids 聚类 (PAM: Partitioning Around Medoids)

#### 原理

将 64 个候选地点聚为 8 类，类中心（medoid）必须是真实数据点（即必须是 64 个地点之一），这恰好满足"场地从参赛单位中选取"的约束。

**标准 PAM 算法**：
```
1. 随机选 8 个初始 medoid
2. BUILD 阶段: 贪心选择使总距离最小的 8 个初始点
3. SWAP 阶段:
   for 每对 (medoid i, 非 medoid j):
       计算用 j 替换 i 后的总距离变化
       若总距离下降，执行交换
   直到无改进
```

#### 两步适配（聚类 → 指派）

阶段一：PAM 聚类得到 8 个 medoid（场地）
阶段二：将 16 个小组分配到场地
- 简单版：每组去距离和最小的场地，但需要容量约束（每场地 2 组）
- ILP 版：固定场地后，用 ILP 求解最优指派

#### 问题

标准 K-medoids 的目标是：每个点到其类中心的距离之和。但我们的"点"是 64 支球队，它们已经分在了 16 个组里——组内 4 队必须去同一个场地。

直接对 64 队聚类可能把同组球队分到不同场地，导致不可行。需要对"16 个组"聚类（每个组的"位置"取组内 4 队的地理中心）。

#### 复杂度

- PAM：O(k(n-k)² × d)，其中 n=16(组)，k=8，即 O(8×8²×d) ≈ 每次迭代 ~500 次距离计算
- 比 ILP 快很多，但解质量略差

### 3.3 模拟退火 (Simulated Annealing)

#### 原理

概率性接受劣解，跳出局部最优。灵感来自固体退火的物理过程。

#### 状态编码

二元组 `(venues, assignment)`：
- `venues`: 长度为 8 的列表，每个元素是 0-63 的索引（8 个场地）
- `assignment`: 长度为 16 的列表，`assignment[g]` = 场地索引

约束：每场地恰好被分配到 2 个组（assignment 由 venues 唯一决定时自动满足）。

#### 邻域操作

1. **替换场地**：随机将某个已选场地替换为一个未选场地，重新指派
2. **交换指派**：两组互换指派场地（保持每场地 2 组）
3. **三组轮换**：三个组间循环互换场地

#### 算法

```
1. 初始解: 启发式贪心选址 + 贪心指派
2. T = T_max (初始温度), cooling_rate = 0.95
3. while T > T_min:
   3.1 从当前解 S 生成邻域解 S'（随机选一种邻域操作）
   3.2 ΔE = cost(S') - cost(S)
   3.3 if ΔE < 0:
       接受 S = S'
       if cost(S) < best_cost: 更新最优解
   3.4 else:
       以概率 exp(-ΔE / T) 接受 S = S'
   3.5 T = T × cooling_rate
4. 返回最优解
```

#### 参数

| 参数 | 建议值 | 影响 |
|------|--------|------|
| T_max | 初始解的 cost × 0.1 | 太大会接受太多劣解，太小退化为局部搜索 |
| T_min | 0.01 | 太小浪费计算时间 |
| cooling_rate | 0.90 ~ 0.99 | 越接近 1，搜索越充分但越慢 |
| 每温度迭代数 | 100 ~ 500 | 马尔可夫链长度 |

#### 优势与局限

| 优势 | 局限 |
|------|------|
| 理论保证收敛到全局最优（时间无限） | 参数调优困难 |
| 实现直观，不需要 ILP 求解器 | 无最优性 gap 保证 |
| 可以嵌入任何约束（如地级市优先） | 运行时间不稳定 |
| 并行化：多条马尔可夫链同时跑 | 冷却速度过快会陷入局部最优 |

### 3.4 遗传算法 (Genetic Algorithm)

#### 原理

维护一个解的"种群"，通过选择、交叉、变异模拟自然进化。

#### 染色体编码

方案 A：**直接编码**
```
[venue_1, venue_2, ..., venue_8 | assign_g1, assign_g2, ..., assign_g16]
```
其中 assign_gi ∈ {1..8}，表示组 i 被指派到第几个场地。

约束处理：
- 交叉/变异后需要修复：确保恰好 8 个不同场地 + 每个场地恰好 2 个 assign

方案 B：**排列编码 + 解码**
```
perm = 一个 0..63 的排列的前 8 个 → 8 个场地
assign_perm = 0..15 的排列 → 每 2 个位置为一个场地的组
```

#### 遗传操作

**选择**：锦标赛选择（随机选 k=3 个个体，保留最优者）

**交叉**：
- 场地部分：两点交叉（注意去重）
- 指派部分：部分匹配交叉 (PMX)

**变异**：
- 场地变异：随机替换一个场地为未选场地
- 指派变异：随机交换两个组的场地

**精英保留**：每代保留前 5% 的个体直接进入下一代。

#### 参数

| 参数 | 建议值 |
|------|--------|
| 种群大小 | 50 ~ 200 |
| 代数 | 100 ~ 500 |
| 交叉概率 | 0.7 ~ 0.9 |
| 变异概率 | 0.05 ~ 0.15 |
| 精英数 | 种群大小 × 0.05 |

#### 优势与局限

| 优势 | 局限 |
|------|------|
| 天然并行（种群独立评估） | 参数多，调参空间大 |
| 适合多目标优化 | 速度通常慢于 SA |
| 维护解的多样性，不易早熟 | 约束修复逻辑复杂 |
| 可产生多个近似最优解供决策者选择 | 无最优性保证 |

### 3.5 变邻域搜索 (Variable Neighborhood Search, VNS)

#### 原理

系统性地在多个邻域结构之间切换，利用"一个局部最优解可能不是另一个邻域结构下的局部最优"这一事实。

#### 邻域结构

| 邻域 N_k | 操作 | 规模 |
|----------|------|------|
| N₁ | 交换一组场地（1换1） | 8 × 56 = 448 |
| N₂ | 交换两组场地（2换2） | C(8,2) × C(56,2) ≈ 43120 |
| N₃ | 交换三组场地（3换3） | 很大 |
| N₄ | 交换两个组的指派场地 | C(16,2) = 120 |
| N₅ | 将某组的指派换到一个空位场地，同时移出"受害组" | 可变 |

#### 算法

```
1. 初始解 x（启发式贪心）
2. 定义 K_max 个邻域结构 {N_1, ..., N_Kmax}
3. while 未达停止条件:
    k = 1
    while k <= K_max:
       3.1 Shaking: 在 N_k 中随机生成 x'
       3.2 Local Search: 对 x' 做局部搜索（如用 N₁ 做爬山），得到 x''
       3.3 Move or Not:
           if cost(x'') < cost(x):
               x = x''
               k = 1      // 回到最小邻域
           else:
               k = k + 1  // 切换到更大邻域
4. 返回最优解 x_best
```

#### 优势

- 确定性算法，结果可复现（如果用固定的 shaking 定点随机）
- 比 SA/GA 参数更少
- 结构化的搜索策略，比随机搜索高效

### 3.6 Benders 分解 (Benders Decomposition)

#### 原理

将问题分解为主问题（选址）和子问题（指派），利用子问题的对偶信息生成 Benders 割，指导主问题的搜索。

#### 主问题 (MP)

只含选址变量 y_k，用变量 θ 代理子问题的最优指派成本：

```
min  θ
s.t. Σ y_k = 8
     Benders cuts (从子问题累积)
     y_k ∈ {0,1}
```

#### 子问题 (SP)

给定场地集合 {k | y_k = 1}，求解最小费用指派：

```
min  Σ_g Σ_k D(g,k) · z_{g,k}
s.t. Σ_k z_{g,k} = 1       (∀g, 共16个约束) → 对偶变量 u_g
     Σ_g z_{g,k} = 2       (∀k with y_k=1)  → 对偶变量 v_k
     z_{g,k} = 0           (∀k with y_k=0)
     z_{g,k} ≥ 0
```

子问题是一个**最小费用流**（或运输问题），有整数最优解（全单模矩阵），松弛为 LP 即可。

#### Benders 割

设子问题最优对偶解为 (u*, v*)，则生成割：

$$
\theta \ge \sum_g u_g^* + 2\sum_k v_k^* \cdot y_k + \sum_k \left(\min(0, \text{reduced cost of unused venues})\right)
$$

对于 y_k = 0 的场地，其对偶信息也贡献到割中（需要计算所有 64 个场地的 reduced cost）。

#### 算法

```
1. UB = +∞, LB = -∞
2. 初始 y（启发式贪心），可选空割集
3. while UB - LB > ε:
   3.1 求解子问题 SP(y) → 得到最优指派成本 Q(y) 和对偶解
   3.2 更新上界: UB = min(UB, Q(y))
   3.3 生成 Benders 割，加入主问题 MP
   3.4 求解主问题 MP → 得到新的 y' 和 θ
   3.5 更新下界: LB = θ
   3.6 y = y'
4. 返回 UB 对应的解
```

#### 优势与局限

| 优势 | 局限 |
|------|------|
| 分解为小问题，结构化求解 | 64 选 8 规模下，变量不多，分解优势有限 |
| 提供上下界和最优性 gap | 实现复杂（需手写对偶/割生成） |
| 子问题是网络流，有快速算法 | 每轮需解 ILP 主问题，小规模下反而不如直接 ILP |
| 经典方法，CPLEX/Gurobi 内部已实现 | 在 64→8 的问题上杀鸡用牛刀 |

### 3.7 基于网络流的算法

#### 3.7.1 最小费用最大流 (Min-Cost Max-Flow)

**当场地已知时的指派问题**：

构图：
```
       cap=1, cost=0        cap=1, cost=D(g,v)        cap=2, cost=0
SOURCE ──────────────→ [g₁...g₁₆] ──────────────────→ [v₁...v₈] ──────────────→ SINK
                       (每组节点)                       (场地节点)
```

这是一个标准的运输问题。由于约束矩阵全单模（totally unimodular），LP 松弛自动得到整数解。

**求解**：用 SSP (Successive Shortest Path) 算法，复杂度 O(F × E log V) = O(16 × (16×8) log 24) ≈ 很快。

**但场地未知时**，需要外层枚举，因为有 C(64,8) 种场地组合，不能暴力。

#### 3.7.2 网络流 + 列生成

如果有 64 个候选场地，对每个场地 k 定义一个"模式"：该场地是否被选中、被分配哪 2 个组。

列生成在设施选址中经典的用法是：
- **主问题**：选择 8 个"场地-组对"模式
- **定价子问题**：寻找负 Reduced Cost 的新模式

但本题由于规模不大，列生成也是过度设计。

#### 3.7.3 b-匹配视角

将问题建模为二分图上的 **b-匹配** (b-Matching)：

- 左节点 16 个，b(g) = 1（每组去一个场地）
- 右节点 64 个，b(k) ∈ {0,2}（选中则容量=2，否则=0）
- 边权 = D(g,k)

目标是找最小权 b-匹配，且满足 Σ b(k)=16（即恰好 8 个场地被激活且满容量）。

这可以转化为 **最小权完美匹配**（通过拆分节点）：
- 将每个右节点 k 拆成 2 个副本 k₁, k₂，容量各 1
- 添加约束：k₁ 和 k₂ 必须"同时激活或同时关闭"

右侧的 64 个节点 × 2 个副本 = 128 个右节点（但最多 8×2=16 个同时激活）。

这与原 ILP 本质上等价，只是用匹配语言重新表述。

## 6. 结论与建议

### 当前最优选择

在 64→8 的规模下，**ILP 联合优化已经是最佳方案**：
- 代码最简洁（[q3_venue.py](q3_venue.py)，核心 ILP 约 60 行）
- 解质量最高（精确最优）
- 运行时间可接受（60秒）

### 值得实现的最有竞争力替代方案

按实现优先级排序：

1. **拉格朗日松弛** — 最具理论价值的补充
   - 提供最优性下界，量化 ILP 解的质量
   - λ_k 可解释为场地"影子价格"，论文中可做经济分析
   - 子问题分解后各组独立，适合做分布式求解的论证

2. **变邻域搜索 (VNS)** — 最具实用价值的近似算法
   - 不需要 ILP 求解器（纯 Python/numpy 实现）
   - 结构化的搜索策略，比 SA/GA 更稳健
   - 可在论文中与 ILP 对比，展示近似算法的效率

3. **模拟退火** — 最简单的手写全局搜索
   - 约 150 行代码，1 小时内可实现
   - 适合嵌入额外软约束（如场地级别偏好）

### 不建议在当前规模下实现的

- **Benders 分解**：过度设计，子问题是小 LP，Benders 割的加速效果不足以抵消框架开销
- **列生成**：同样过度设计，64→8 的规模不需要这种工业级方法
- **单纯网络流**：流量模型无法处理"选场地"的决策，只能作为内层指派子程序

## 7. 拉格朗日松弛实现结果

**文件**：[q3_lagrangian.py](q3_lagrangian.py)（~450 行）

### 7.1 算法架构

实现采用三层分析结构：

**Layer 1 — 有效下界 (Valid LB)**：
- 松弛"恰好 8 个场地"的约束，退化为运输问题
- 问题：min Σ D(g,k) x(g,k), s.t. Σ_k x=1, Σ_g x≤2, x∈{0,1}
- 由于约束矩阵全单模 (totally unimodular)，贪心算法给出精确最优
- 贪心策略：按 D(g,k) 升序排序所有 (组, 场地) 对，依次选取可行的最小边
- 这提供了**严格的数学下界**——任何可行解都不可能低于此值

**Layer 2 — 拉格朗日启发式 (UB)**：
- 松弛"每场地恰好 2 组"容量约束
- 子问题退化为 p-median 选址 (64 选 8)，仍是 NP-hard
- 用多起点贪心 + 增量局部搜索近似求解子问题
- 次梯度优化迭代 200 轮，自适应 Polyak 步长
- 从松弛解修复得到原问题可行解

**Layer 3 — 影子价格**：
- 收敛后的 λ_k 解释为场地容量约束的边际价值
- |λ_k| 越大 → 放宽该场地容量能减少更多总距离

### 7.2 并行调度

使用 `asyncio` + `ThreadPoolExecutor` 并行调度：

```
asyncio 事件循环
  ├── ThreadPoolExecutor (10 workers)
  │   ├── Worker 1: λ_init=zeros,  rule=polyak
  │   ├── Worker 2: λ_init=zeros,  rule=1/t
  │   ├── Worker 3: λ_init=random, rule=polyak
  │   ├── ...
  │   └── Worker 10: λ_init=cost_based, rule=1/t
  └── 合并: 取所有 worker 的最优结果
```

Numpy 运算释放 GIL 后，10 个线程可获得接近线性的加速比。

### 7.3 运行结果

以蛇形分组 (scheme_b_serpentine) 为例（PuLP 不可用时的回退方案）：

```
┌──────────────────────────────────────────────────────┐
│  简单下界 (忽略所有约束):        7879 km              │
│  有效下界 (松弛场地数=8):        7883 km              │
│  拉格朗日可行解 (UB):            8109 km              │
│  [参考] ILP最优  (不同分组):     7617 km              │
└──────────────────────────────────────────────────────┘

最优性保证:
  有效下界 → UB:  gap ≤ 2.8%
  任何可行解不可能 < 7883 km
```

**分析**：
- 有效下界 7883 km 与可行解 8109 km 之间的 gap 仅 2.8%，说明拉格朗日启发式找到了质量很高的解
- 简单下界 (7879 km) 与有效下界 (7883 km) 仅差 4 km，原因是"不限场地数"的松弛已经很弱——多开放几个场地后，几乎每组都能选到自己最近的场地
- 参考 ILP 最优值 7617 km 使用了不同的分组方案 (ILP 均衡分组)，数值不可直接比较

**收敛过程**：
```
初始 |γ|: 5.3  →  最终 |γ|: 2.0
初始 UB:  8652 km  →  最终 UB:  8109 km  (改进 543 km)
迭代次数: 200
```

**影子价格分析**：
| 场地 | λ (km) | 经济解释 |
|------|--------|---------|
| 嵊泗县 | +206 | 海岛边缘，容量最"值钱" |
| 岱山县 | +123 | 海岛，多一个组位可省 123 km |
| 庆元县 | +111 | 浙南边陲，供给不足 |
| 武义县 | -90 | 浙中核心，使用充分 |
| 嵊州市 | -82 | 位置适中，容量相对富余 |

λ 分布：[-90, -45, -1, 35, 206] (min / Q1 / median / Q3 / max)

### 7.4 方法论的局限与说明

**关键局限**：拉格朗日子问题 (p-median) 是 NP-hard 的，因此我们只能近似求解子问题，导致：

1. 计算出的 L(λ) 不是在该 λ 下的严格最小值 → `heuristic_lb` 不是有效下界
2. 但次梯度方向仍然是有效的下降方向 → λ 更新仍然有意义
3. 真正的有效下界来自 Layer 1 的松弛运输问题

**改进方向**：
- 如果安装 PuLP，可用 LP 松弛代替贪心子问题求解，得到严格更紧的下界
- 或者用 branch-and-bound 精确求解 p-median 子问题（问题规模 64 选 8，理论上可解）

## 8. 多维度成本模型对比 (v2)

**文件**：[q3_venue_v2.py](q3_venue_v2.py)

### 8.1 动机

原始算法仅使用 Haversine 大圆距离度量成本。然而在真实的赛事组织中，"距离"远非唯一因素：
- 浙江地形复杂（山地、海岛），直线距离 100 km 可能对应 3 小时公路车程；
- 铁路网络显著改变可达性格局，杭州、金华等枢纽站点具有天然优势；
- 人口集中的城市（杭州、宁波、温州）拥有更多球迷，其出行成本应被赋予更高权重。

[conditions_q3.zip](conditions_q3/) 提供了真实交通数据，使得多维度成本建模成为可能。

### 8.2 数据来源

| 数据 | 文件 | 说明 |
|------|------|------|
| 公路旅行时间 | `交通/travel_time_matrix_minutes.csv` | 64×64 矩阵，自驾/大巴 (min) |
| 铁路旅行时间 | `交通/railway_travel_time_matrix_minutes.csv` | 64×64 矩阵，高铁/普速 (min) |
| 可支配收入 | `可支配收入/可支配收入.json` | 65 条区县人均可支配收入 (元) |
| 常住人口 | `常住人口/常住人口（万人）.json` | 65 条区县常住人口 (万人) |

> **注**：数据中 "景宁畲族自治县" 与参赛队名 "景宁县" 指同一地点，代码中通过 `_resolve_name()` 双向映射解决。

### 8.3 五种成本模型

对每种模型，构建组成本矩阵 D(g,k) = 组 g 内 4 队到场地 k 的成本之和，然后用 ILP 联合优化选址 + 指派。

| 编号 | 模型 | 单位 | 含义 |
|------|------|------|------|
| **A** | `haversine` | km | 大圆距离（基准） |
| **B** | `road_time` | min | 公路旅行时间 |
| **C** | `railway_time` | min | 铁路旅行时间 |
| **D** | `combined_time` | min | 0.5×(公路 + 铁路) 平均 |
| **E** | `fan_weighted` | 万人·min | 公路时间 × 源城市人口，最小化"球迷总旅行负担" |

模型 E 的构造思想：一支来自人口 W_i 万人的球队，其球迷前往场地 k 的"社会总时间成本"为 travel_time(i,k) × W_i。这是从"球队出行"到"球迷出行"的视角转换。

### 8.4 运行结果（ILP 精确最优，同一分组方案）

使用 ILP 均衡分组 (scheme_d_ilp_balanced)，C1/C2/C3 全部满足。

| 模型 | 总成本 | 单位 | 平均每队 | 最远单队 | Haversine 参考 |
|------|--------|------|----------|----------|---------------|
| haversine | 7,648 | km | 119.5 | 337 km | 7,648 km |
| road_time | 8,154 | min | 127.4 | 356 min | 8,317 km |
| railway_time | 6,366 | min | 99.5 | 275 min | 8,273 km |
| combined_time | 7,542 | min | 117.8 | 270 min | 8,187 km |
| fan_weighted | 546,399 | 万人·min | 8,537.5 | 36,077 万人·min | 8,985 km |

**关键观察**：

1. **铁路模型总时间最短**（6,366 min），比公路模型少 22%。浙江省高铁网络覆盖较好，铁路出行在时间效率上优势明显。

2. **所有交通时间模型的 Haversine 参考距离均高于纯 Haversine 模型**（+7.0%~+17.5%）。这意味着：优化真实旅行时间比优化直线距离更"费距离"——交通网络扭曲了空间，使得"时间最优" ≠ "距离最优"。

3. **球迷加权模型在 Haversine 距离上表现最差**（+17.5%），因为它主动牺牲距离来换取"人口就近"——将场地向人口大市（杭州、宁波、温州）集中。

### 8.5 选址一致性分析

各模型选出的 8 个场地集合之间的 Jaccard 交集：

```
         haversine ∩ road_time      = 1/8
         haversine ∩ railway_time   = 2/8
         haversine ∩ combined_time  = 1/8
         haversine ∩ fan_weighted   = 0/8   ← 完全不同!

         road_time ∩ railway_time   = 3/8
         road_time ∩ combined_time  = 4/8
         combined_time ∩ fan_weighted = 4/8
```

**解读**：

- **Haversine 与交通时间模型交集极低**（0-2/8），说明纯距离模型选出的场地在网络拓扑下并非最优。例如，Haversine 选出了余姚市、新昌县、东阳市等"几何中心"型场地，但交通模型倾向于选择杭州、金华、义乌等"网络枢纽"型场地。

- **公路和铁路模型之间有 3/8 交集**，说明两类交通网络有一定重叠但仍有各自最优选择。

- **combined_time 与其他交通模型交集较高**（4/8），作为折中方案可能最稳健。

**高频场地**（被 ≥3 个模型选中）：

| 场地 | 被选次数 | 特征 |
|------|----------|------|
| 杭州市 | 4/5 | 省会，铁路枢纽，公路中心 |
| 金华市 | 4/5 | 浙中交通枢纽 |
| 义乌市 | 4/5 | 高铁枢纽，经济中心 |
| 永康市 | 3/5 | 浙中工业重镇 |

杭州市、金华市、义乌市是所有成本模型都认可的"必选场地"（除了纯 Haversine 模型）。它们在浙江省交通网络中处于枢纽地位。

### 8.6 模型选择建议

| 赛事目标 | 推荐模型 | 原因 |
|----------|----------|------|
| 最小化球队总旅行距离 | A (`haversine`) | 最直观的公平性度量 |
| 最小化球队实际旅行时间 | D (`combined_time`) | 公路+铁路折中，避免单一交通方式偏差 |
| 最小化球迷出行总负担 | E (`fan_weighted`) | 让更多球迷就近观赛，体现社会公平 |
| 论文展示"考虑实际交通" | B + C 对比 | 展示公路/铁路对选址的不同影响 |
| 稳健决策 | A + D + E 综合投票 | 取三个模型均高频的场地（杭州、金华、义乌等） |

### 8.7 核心结论

1. **不同成本模型会导致几乎完全不同的场地选择**（Jaccard ≤ 2/8 for Haversine vs 交通模型）。这表明仅凭经纬度距离做决策存在系统性偏差——交通网络拓扑是一个不可忽视的因素。

2. **铁路网络显著优于公路**（总时间短 22%），浙江省的高铁网络为赛事分散举办提供了可行性支撑。

3. **球迷加权模型揭示了公平性的另一维度**：最小化"球队旅行距离"不等同于最小化"球迷出行负担"。后者需要将人口分布纳入优化目标。

4. **建议实际赛事组织采用多模型综合决策**：以 combined_time 为主体方案，以 fan_weighted 校验公平性，以 Haversine 距离作为最终可达性兜底检查。

### 8.8 方法对比：v1 vs v2
